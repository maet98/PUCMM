\hypertarget{md_hellopintos_autotoc_md1}{}\doxysubsubsection{Locus para el desarrollo de la aplicación solicitada}\label{md_hellopintos_autotoc_md1}
Para el desarrollo de la aplicación se uso el siguiente Locus\+:

\begin{center}(\$\+H\+O\+ME)/\+P\+I\+N\+T\+O\+S-\/\+Ubuntu/pintos/src/tests/threads/\end{center} 

Se uso este locus ya que aquí se encuentran varios archivos de test lo cuales se pueden correr con pintos al especificar el alias que se le ha sido asignado, a la hora de ejecutar pintos.\hypertarget{md_hellopintos_autotoc_md2}{}\doxysubsubsection{Producción de ejecutable y documentación}\label{md_hellopintos_autotoc_md2}
Para producir el ejecutable de la aplicación se necesita situar en el directorio {\itshape src/threads/} una vez localizados en este directorio basta con escribir en la consola\+:

\begin{center}make clean; make all\end{center} 

Lo cual invocara al makefile y tendrá listo pintos para su ejecución, y para ejecutar la aplicación en el mismo directorios se escribe en la consola\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{pintos -\/-\/ run hello-\/pintos}
\end{DoxyCode}


Para producir la documentación debemos usar el archivo {\itshape Doxypintos.\+doxyfile} y localizarlo en el mismo directorio en que se encuentra \+\_\+/src\+\_\+ (\+\_\+/pintos\+\_\+) y con el generador doxygen simplemente se escribe\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{doxygen Doxypintos.doxyfile}
\end{DoxyCode}


Esto creara un directorio {\itshape doc}, donde se encontrara toda la documentación del proyecto pintos con la aplicación hello-\/pintos.
